# RAG PDF Q&A — Setup & Run Guide
**Samsung GenAI Program · Bonus practice tool — not scored, not submitted**

A hands-on RAG (Retrieval-Augmented Generation) app — upload real PDFs, build a real vector index, and ask real questions grounded in your documents. Runs 100% on your own laptop via Ollama — no API key, no cost, no internet needed once set up.

> **Already set up Ollama for Day 2's Prompt Playground or Day 3's Interrogation Room?** You likely already have `llama3.2:1b` — skip to Step 2 and just pull `nomic-embed-text` too.

---

## Step 1 — Install Ollama (skip if already done)
Download and install from [ollama.com](https://ollama.com) if you haven't already for a previous day's tool.

## Step 2 — Pull both models
Open Command Prompt and run:
```bash
ollama pull nomic-embed-text
ollama pull llama3.2:1b
```
`nomic-embed-text` is ~274MB, `llama3.2:1b` is ~1.3GB — start these now if you don't already have them, they take a few minutes over classroom WiFi.

## Step 3 — Confirm Ollama is running
```bash
ollama list
```
You should see both models listed. If you get a connection error instead, run `ollama serve` in a separate Command Prompt window (leave it running).

## Step 4 — Confirm Python and install dependencies
Open Command Prompt in the `rag-project` folder and run:
```bash
pip install -r requirements.txt
```
This installs LangChain, ChromaDB, and Streamlit — no API key packages needed.

## Step 5 — Run the app
```bash
streamlit run app.py
```
Opens automatically at **http://localhost:8501**.

---

## Using it
1. Upload one or more PDFs in the sidebar (try the included `Corporate_HR_Policy_Document.pdf` or `nasscom-hr-manual.pdf` first).
2. Click **Build Index** — this chunks and embeds your PDF locally via `nomic-embed-text`.
3. Ask a question in the main area — the answer is generated by `llama3.2:1b`, grounded in the chunks retrieved from your PDF.
4. Expand **Sources** to see exactly which chunks the answer came from.

## Troubleshooting

| Problem | Fix |
|---|---|
| `Error: listen tcp 127.0.0.1:11434: bind: ... already permitted` when running `ollama serve` | This means Ollama is **already running** — not an error. Run `ollama list` to confirm, then just start the app. |
| App shows a wall of `transformers`/`torchvision` errors in the terminal | Harmless — Streamlit's file-watcher tripping over an unrelated cached import from an old session. Fully stop the app (Ctrl+C) and run `streamlit run app.py` fresh — a clean process won't show this. |
| "Indexing failed" or dimension-mismatch error | Click **Clear Index** in the sidebar, then re-upload and rebuild. This can happen if `chroma_db/` was ever built with a different embedding model. |
| Answers seem to ignore your PDF | Check the sidebar — does it say "Index ready — N chunks"? If it says "No index yet," you need to Build Index first. |
